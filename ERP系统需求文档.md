# ERP系统需求文档（零售行业版 V1.1）

## 1. 文档信息
- 项目名称：连锁零售ERP系统建设
- 文档版本：V1.1（零售行业版）
- 编写日期：2026-03-06
- 适用对象：业务负责人、产品经理、研发、测试、实施、运维

## 2. 项目背景与目标
当前零售业务存在“门店、仓库、电商平台数据分散，商品/价格/促销口径不统一，库存不准导致缺货与滞销并存”的问题，影响销售与利润。建设ERP系统的目标如下：

1. 打通“商品-采购-仓配-门店/电商销售-会员-财务”全链路，实现业务与财务一体化。
2. 建立统一商品主数据、价格体系、促销规则和库存口径，支撑全渠道协同。
3. 提升库存周转效率，降低缺货率与滞销率，提升单店人效与坪效。
4. 建立经营分析体系，支持总部对门店和渠道的精细化管理。

## 3. 业务范围与边界

### 3.1 本期范围（MVP）
1. 主数据管理（商品、分类、条码、供应商、门店、仓库、会员）
2. 商品中心（商品档案、价格管理、生命周期管理、上下架）
3. 采购与补货（请购、采购订单、到货收货、门店补货建议）
4. 库存与仓配（仓库库存、门店库存、调拨、盘点、损耗、效期）
5. 门店POS（收银、挂单、退换货、交接班、日结）
6. 全渠道订单（电商订单同步、配货、发货、退货入库）
7. 会员与营销（会员等级、积分、优惠券、促销活动）
8. 财务结算（应收应付、门店日结对账、凭证接口、月结）
9. 审批流与权限管理（角色权限、流程配置、操作审计）
10. 报表与经营看板（销售、毛利、库存、会员、门店绩效）

### 3.2 暂不纳入范围（后续阶段）
1. 智能动态定价与AI选品
2. 高级供应链网络优化（多级仓智能调拨）
3. 跨境零售多币种税务体系
4. 深度营销自动化（CDP/私域营销编排）

## 4. 角色与职责
- 系统管理员：组织参数、权限与基础配置维护
- 商品运营：商品建档、分类、上下架、价格策略维护
- 采购专员：采购下单、到货跟踪、供应商协同
- 供应链计划员：补货策略、调拨计划、缺货预警处理
- 仓库管理员：收货、上架、拣配、盘点、损耗处理
- 门店店长：门店库存、人员班次、日结审核、经营分析
- 收银员：POS收银、退换货、交接班
- 电商运营：平台订单处理、发货与售后跟进
- 财务会计：对账、核销、凭证审核、月结
- 管理层：渠道经营分析、利润监控、审批决策

## 5. 核心业务流程需求

### 5.1 采购到收货（P2R）
1. 总部/门店发起请购，系统结合安全库存与销量预测生成采购建议。
2. 审批通过后生成采购订单，支持分批到货、部分收货、差异处理。
3. 收货后自动更新可用库存，触发应付暂估并记录批次/效期信息。
4. 采购退货需关联原到货单，形成库存与应付冲减闭环。

### 5.2 补货与调拨（Replenishment）
1. 系统按门店销量、库存天数、在途量生成补货建议单。
2. 支持“仓到店”配送与“店到店”调拨两种模式。
3. 调拨全流程可追踪（出库、在途、签收、差异处理）。
4. 缺货门店自动预警并推送供应链计划员与店长。

### 5.3 门店销售到日结（POS2Close）
1. 门店POS支持条码扫描、会员识别、促销自动计算、组合支付。
2. 销售完成后实时扣减门店库存并记录毛利数据。
3. 退换货支持原单/无单模式，并自动回写库存与财务冲销。
4. 班次结束进行交接班，门店每日完成日结对账（系统金额=实收金额）。

### 5.4 全渠道订单履约（O2F）
1. 电商平台订单自动同步至ERP订单中心，统一库存占用。
2. 系统按就近仓/店策略分配履约节点并生成拣货任务。
3. 发货后回传物流单号与轨迹，支持签收状态同步。
4. 售后退货支持质检分流（可二次销售/报损）并完成退款对账。

### 5.5 会员营销到复购（M2R）
1. 会员注册与标签统一管理，支持门店与线上渠道身份打通。
2. 促销规则支持满减、折扣、赠品、优惠券叠加策略配置。
3. 积分按规则发放与扣减，支持积分抵现与过期提醒。
4. 通过复购率、客单价、会员贡献占比评估活动效果。

### 5.6 月结与经营分析
1. 月末完成库存结转、销售成本核算、应收应付账龄更新。
2. 财务锁定会计期间后，业务单据仅允许走红字/冲销流程。
3. 自动输出门店、区域、渠道、品牌维度经营报表。

## 7. 非功能需求

### 7.1 性能与容量
1. POS收银主流程（扫码到支付完成）95%请求 ≤ 1 秒。
2. 高峰期（促销活动）支持不少于 2,000 并发交易请求。
3. 报表查询（千万级交易数据）15 秒内返回首屏结果。

### 7.2 可用性与稳定性
1. 年度可用性目标：99.95%（营销大促期间核心功能可用）。
2. 支付、扣减库存、订单确认等关键链路必须支持幂等处理。
3. 具备灰度发布、限流降级与快速回滚能力。

### 7.3 安全与合规
1. 用户密码加密存储，支持账号锁定与异常登录告警。
2. 会员手机号、证件号等隐私数据脱敏与最小权限访问。
3. 全链路HTTPS与关键接口签名校验，防重放攻击。
4. 审计日志保留至少 12 个月，并支持按门店/用户检索。

### 7.4 运维与灾备
1. 每日全量备份 + 每小时增量备份。
2. RPO ≤ 1 小时，RTO ≤ 2 小时。
3. 建立交易成功率、库存同步延迟、接口失败率监控告警。

## 8. 数据需求

### 8.1 核心数据实体
- 主体组织：总部、区域、门店、仓库、用户、角色
- 商品数据：SPU/SKU、分类、条码、品牌、价格、促销规则
- 交易数据：采购单、收货单、调拨单、POS小票、订单、退货单
- 会员数据：会员档案、积分账户、优惠券账户、标签
- 财务数据：日结单、应收应付、收付款、凭证

### 8.2 数据治理要求
1. 商品、供应商、门店主数据新增/变更必须审批。
2. 统一编码规则（对象前缀+日期+流水号）与唯一性校验。
3. 统一单据状态机（草稿/审批中/已审批/执行中/已完成/已关闭）。
4. 建立数据质量监控（重复商品、异常价格、库存负数）告警。

## 9. 集成需求
1. 支付渠道：微信支付、支付宝、银行卡受理与对账接口。
2. 电商平台：订单、库存、发货、售后状态双向同步。
3. 门店硬件：POS终端、扫码枪、小票打印机、钱箱。
4. 物流平台：运单号、轨迹、签收、逆向物流信息同步。
5. 财务系统：科目同步、凭证下发、回执状态回传。
6. 企业IM：待办审批、库存预警、销售战报通知。

## 10. 里程碑与交付

### 10.1 里程碑建议
1. 需求蓝图与原型确认：2~3 周
2. MVP开发与联调：10~12 周
3. UAT与门店试点：3~4 周
4. 全面上线与稳定期：2~4 周

### 10.2 MVP验收标准
1. P0功能全部上线并通过UAT。
2. 门店POS交易成功率 ≥ 99.95%，关键流程端到端可追溯。
3. 库存准确率 ≥ 98%，缺货率较现状下降 ≥ 20%。
4. 门店日结自动对账成功率 ≥ 99%，异常单据可闭环处理。
5. 权限审计符合内控要求，无高危安全漏洞。

## 11. 风险与待确认事项
1. 商品与促销规则复杂度高，是否需要先约束促销策略模板。
2. 历史主数据（商品、会员、库存）质量是否满足迁移标准。
3. 电商平台接口稳定性与变更频率是否可控。
4. 门店网络波动场景下，是否需要POS离线交易能力。
5. 多门店差异化运营策略（价格/促销）是否统一治理。

## 12. 附录：术语说明
- POS：Point of Sale，门店收银系统
- SPU/SKU：标准化商品与最小库存单位
- O2F：Order to Fulfillment，订单到履约流程
- P2R：Procure to Receive，采购到收货流程
- RBAC：Role-Based Access Control，基于角色的权限控制
